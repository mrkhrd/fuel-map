# fuel-map

Карта наличия топлива на АЗС: данные [toplivo.tbank.ru](https://toplivo.tbank.ru)
объединяются с [sberazs.ru](https://sberazs.ru), маршруты по реальным дорогам (OSRM).

## Возможности

- АЗС на карте (Leaflet + OSM), подгрузка по видимой области; цвет маркера —
  объединённый статус двух источников (Сбер учитывается только как свежий
  позитивный сигнал — устаревшие данные не перекрывают Т-Банк)
- Станции Сбера сопоставляются с Т-Банком по координатам (до 150 м)
- Бейдж у маркера — по строке на банк: время последней оплаты (зелёный < 3 ч,
  серый > 24 ч), у Сбера — число оплат; цветные чипы наличия по видам топлива
- Попап: секции Т-Банк и Сбер — наличие по видам топлива, лимиты литров,
  оплаты, уверенность оценки
- Фильтры: по виду топлива (OR двух источников: АИ-92…АИ-100, ДТ, пропан, метан)
  и по статусу (есть / возможно / нет / нет данных)
- Маршрут (TSP) по видимым или выбранным станциям (ctrl+клик — добавить сразу,
  либо попап: Старт / Точка / Финиш); матрица времени и геометрия — OSRM,
  решатель nearest-neighbour + 2-opt с фиксацией старта/финиша; fallback
  по прямой при недоступности роутера; ссылка «открыть в Я.Картах»

## Запуск

Rust-хост (~240 КБ, `index.html` вшит при сборке, TLS из ОС):

```
build-rust.bat                        # → target\release\fuel-host.exe
fuel-host.exe [порт] [host=ip ...]    # порт по умолчанию 8000
```

`host=ip` закрепляет адрес внешнего API (аналог docker `extra_hosts`) — нужно
для sberazs.ru, который переехал за DDoS-Guard и отдаёт JS-челлендж вместо JSON:

```
fuel-host.exe 8000 sberazs.ru=213.171.31.57
```

Альтернативы: `python server.py` или PyInstaller-сборка `build.bat`
(~9 МБ, `dist\fuel-map-server.exe`).

Сервер — статика + прокси `/api/*`, `/sber/*`, `/osrm/*` (у внешних API нет
CORS-заголовков). `index.html` рядом с exe имеет приоритет над встроенным.
Все запросы клиента и походы к внешним API логируются с таймингами.

## Релизы и Docker

Каждый push в `main` автоматически повышает patch-версию (тег `vX.Y.Z`),
публикует GitHub Release с `fuel-map-vX.Y.Z-win64.zip` и docker-образ;
версия зашита в бинарник и печатается при старте:

```
docker run -d -p 8000:8000 --add-host sberazs.ru:213.171.31.57 ghcr.io/mrkhrd/fuel-map:latest
docker run -d -p 8123:8123 ghcr.io/mrkhrd/fuel-map:latest 8123   # свой порт
```

В compose пин задаётся через `extra_hosts: ["sberazs.ru:213.171.31.57"]`.

## Автозапуск на Windows

```powershell
netsh advfirewall firewall add rule name="fuel-map" dir=in action=allow protocol=TCP localport=8000
schtasks /Create /TN "fuel-map" /SC ONSTART /RU SYSTEM /TR "C:\path\to\fuel-host.exe"
schtasks /Run /TN "fuel-map"
```

## Замечания

- Публичный OSRM demo-сервер rate-limited; при недоступности маршрут строится по прямой.
- Сервер без аутентификации и HTTPS — рассчитан на локальную сеть.
